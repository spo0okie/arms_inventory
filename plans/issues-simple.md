## 6. #205 — Поле комментария к изменению для журнала
**Сложность: средняя (~4 ч).**
Поля комментария в таблицах/моделях `*History` уже есть — надо только заполнять.
- Добавить в форму редактирования моделей необязательное поле «комментарий».
- Пробрасывать его в запись `*History` при сохранении.

- Тест: сохранение с комментарием без изменений создаёт запись истории.

## 9. #68 — Отображение линков без повторов
**Сложность: средняя (~4–6 ч), но объём чётко ограничен.**
Сущности «линк» нет — есть привязка порт→порт, поэтому каждый линк выводится дважды
(A↔B и B↔A).
- Дедуп на уровне SQL/выборки (например, брать пары, где `portA.id < portB.id`).
- Не заводить новую сущность в рамках этой задачи (это отдельная бОльшая работа).
- Тест: на паре связанных портов список содержит одну запись линка.
