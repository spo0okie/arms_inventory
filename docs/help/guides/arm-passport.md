# Паспорт АРМ

Паспорт АРМ («Паспорт рабочего места») — документ, формируемый по данным инвентаризации для конкретного АРМ. Открывается со страницы просмотра АРМ по иконке паспорта в правом верхнем углу.

Остальная информация об АРМ ([привязанные ОС](../models/comps/arm_id.md), назначенные лицензии, [связанные документы](../models/contracts.md), [материалы и ЗИП](../models/materials.md#использованиерасход-материалов), когда-либо израсходованные на этот АРМ, [заметки](../models/techs/notepad.md) о жизненном цикле рабочего места) отображается на самой странице просмотра АРМ.

## Шапка

В шапке паспорта отображается:

- Идентификационный номер АРМ
- Блок «Компьютер»: доменное имя основной операционки, модель, [инвентарный номер](../models/techs/num.md) и серийный номер
- Блок «Пользователь»: ФИО, учётная запись и подразделение пользователя

## Софт

### Игнорируемое ПО

Сюда выводится всё обнаруженное на привязанных к АРМ [операционках](../models/comps.md) игнорируемое ПО.
Добавить в паспорт его невозможно.

### Бесплатное ПО

Сюда выводится всё обнаруженное на привязанных к АРМ [операционках](../models/comps.md) бесплатное ПО.
Добавить в паспорт его невозможно.

> Если игнорируемое или бесплатное ПО ранее уже было внесено в паспорт (например, до смены его категории в справочнике), в соответствующей секции появится кнопка «Убрать из паспорта … ПО на <ИМЯ_ОС>» для массовой чистки.

### Неизвестное ПО

Это программные продукты, обнаруженные на привязанных к АРМ [операционках](../models/comps.md), которые не описаны в таблице программных продуктов.
Добавить в паспорт его невозможно, нужно его сначала распознать:

- Если производитель не распознан, то нужно добавить его написание к одному из производителей в базе (кнопка «+»)
  - Откроется окно «Добавить написание производителя», т.к. изначально мы предполагаем, что такой вендор у нас уже есть, мы просто не знали, что он пишется вот таким образом.
  - Если в списке вендоров отсутствует нужный, значит, жмём кнопку «Создать нового производителя».
- Потом надо распознать программный продукт, нажав:
  - кнопку «гаечный ключ», если это наименование нужно добавить к уже известному ПО
  - кнопку «+», если нужно создать новый продукт

### ПО не из реестра

Сюда выводится всё обнаруженное на привязанных к АРМ [операционках](../models/comps.md) ПО, которое не является ни игнорируемым, ни бесплатным (т.е. вполне себе платное), но при этом не входит в реестр согласованного ПО.
Добавить в паспорт его невозможно. Нужно либо сначала внести его в реестр согласованного ПО и потом добавить в паспорт, либо удалить это ПО с машины.

### ПО не из паспорта

(на странице секция называется «На компьютере обнаружено платное ПО не из паспорта»)

Вот это ПО можно добавлять. Это ПО из реестра согласованного ПО, но не внесённое в паспорт. Т.е. для программных продуктов в этой секции два пути:

- внести в паспорт (кнопка «+»), т.к. установка этого ПО на это рабочее место согласована
- удалить это ПО с машины, т.к. его установка на это рабочее место не согласована.

Для массового внесения всего ПО из реестра согласованного ПО есть кнопка «Внести в паспорт ПО на <ИМЯ_ОС>» (для каждой ОС, закреплённой за АРМ).

### Программное обеспечение

Тут выводится ПО, внесённое в паспорт. Если оно обнаружено на машине — будет подсвечиваться зелёным, если пропало — жёлтым.
Тут его можно из паспорта удалить, нажав кнопку «−».

## Аппаратное обеспечение

Выводится табличка с аппаратным обеспечением на основной операционке машины.

- Чтобы добавить компонент в паспорт, надо нажать на кнопку «+» в конце строки компонента
- Чтобы добавить всё оборудование, которое ещё не добавлено, надо нажать оранжевую кнопку «Согласовать оборудование»
- Чтобы убрать компонент из паспорта, надо нажать кнопку «−»
- Есть ещё «карандаш», который позволяет скорректировать отображение компонента, чтобы он выводился не точно так, как его обнаружил скрипт.
  - Изначально эта возможность была нужна, чтобы добавлять компоненты вручную, для добавления телефона/ИБП и т.п., но на текущий момент такие процедуры надо делать через ввод оборудования в инвентаризацию и [прикрепление к АРМ](../models/techs/arms_id.md)
  - Также, нажав редактирование, можно скрыть компонент (если это какое-то незначительное USB-оборудование, которое не должно отсвечивать в паспорте)
  - Можно внести серийные номера компонент, которые не обнаружил скрипт

## Доп. оборудование

Табличка с прикреплённым к АРМ оборудованием (тип, модель, идентификатор, серийный и инвентарный номера). Если ничего не прикреплено — пишется «Отсутствует».
Кнопкой «Внести доп. оборудование» можно завести новое оборудование сразу с привязкой к этому АРМ.

## Подвал

Он же footer.
Сюда выводятся подписанты паспорта:

- Сотрудник службы ИТ (в паспорте — «Сотрудник дирекции ИТ»)
- Руководитель отдела (в паспорте — «Руководитель структурного подразделения»)
- Пользователь (в паспорте — «Сотрудник»)
- Адм.Ответственный (если установлен — отдельной строкой как осуществляющий установку и изменение аппаратно-программной конфигурации)

## Печать

В печать выводится только кусок, начинающийся с большого слова ПАСПОРТ и заканчивающийся подвалом (служебные кнопки и секции с несогласованным ПО не печатаются).
Софт и железо распечатываются только явно внесённые в паспорт.
