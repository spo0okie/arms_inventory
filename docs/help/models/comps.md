# Операционные системы

Ручную инвентаризацию железа ([оборудования](techs.md), [ЗИП и Материалов](materials.md)) дополняет полуавтоматическая инвентаризация Операционных систем.
Есть несколько типов скриптов, собирающих информацию об установленных ОС:

- [Импорт скриптами из Windows](https://github.com/spo0okie/win-inventory) — Сканирует установленное железо, софт, IP адреса — распространяется по операционным системам Windows при помощи GPO
- [Импорт скриптами из Linux](https://github.com/spo0okie/puppet_inventory_scripts) — Сканирует установленное железо, IP адреса — распространяется по операционным системам Linux средствами управления конфигурациями (например Puppet)
- [Импорт скриптами из VMWare](https://github.com/spo0okie/powercli_inventory_scripts) — Сканирует установленное железо, IP адреса, раскладывает операционки по [АРМам](techs.md) согласно расположению ВМ на хостах, работает с одного узла с установленным PowerCLI

## Список

В списке ОС вроде бы и так очевидно все. Можно сортировать (заголовки кликабельны), можно фильтровать по полям ввода (подсказки по фильтрации выводятся при наведении мыши на заголовок).
Цвет ячейки с именем хоста определяется временем с последнего обновления:

- Меньше часа — ярко голубой
- Меньше суток — бледно голубой
- Меньше недели — зеленый (имеет смысл обратить внимание, если ОС предполагается быть включенной ежедневно)
- Меньше месяца — желтый (имеет смысл обратить внимание, если ОС предполагается быть включенной хотя бы раз в неделю)
- Больше месяца — красный (либо ОС загружается только при необходимости, либо с ней что-то не так)

> Из неочевидных вещей: **Поиск софта/железа**
> Если в поле софт вписать текст, то фильтроваться будут те ОС, у которых этот текст встречается в отпечатке софта или в [отпечатке железа](comps/raw_hw.md), т.е. так можно найти ОС с конкретным серийником материнской платы, моделью платы, версией корела и т.п.

## Просмотр

### Шапка

- Рядом с названием и кнопочками редактирования и удаления есть кнопочка со значком входа (стрелка в дверь) — которая может открыть программу удаленного доступа
- Есть строка с версией операционной системы (она включена как отдельный продукт в список ПО)
- Сверху цветной ярлычок со временем, прошедшим с обновления данных (цвет меняется согласно списку цветов выше). Если на него кликнуть, отобразится детальная строка обновления данных, которая состоит из
  - Даты-времени обновления — момент, когда скрипт внес данные об этой ОС в БД
  - Версии скрипта (пишется в скобочках),
    - если версия заканчивается суффиксом **nix**, значит данные собраны [скриптами для Linux/Unix](https://github.com/spo0okie/puppet_inventory_scripts)
    - если версия заканчивается суффиксом **powercli**, значит данные собраны [скриптом для powercli](https://github.com/spo0okie/powercli_inventory_scripts)
    - если пусто — значит ОС завели руками (в определенных случаях такое тоже нужно)
    - в остальных случаях там записана версия [скрипта для Windows](https://github.com/spo0okie/win-inventory)

### IP адреса

Выводит список адресов на всех интерфейсах.
Иногда появляются какие-то технические адреса (виртуальные интерфейсы, туннельные и т.п.), если они отображаются в карте помещений вместо основного адреса, то их можно скрыть (перенести в список игнорируемых IP), а потом при необходимости вернуть обратно. <!-- TODO: проверить — раньше это делалось значками глаза (eye-close/eye-open) прямо в списке IP, в текущем интерфейсе такой значок не найден -->
Можно открыть программу удаленного доступа, кликнув на значок входа (стрелка в дверь)

### Журнал входов

Выписка из 3-х последних записей входа на эту ОС из журнала входов в ПК

## Редактирование

Назначение полей показывают подсказки прямо в форме. Штатно меняются только
Домен и [Имя компьютера](comps/name.md) (там же — как правильно
переименовывать) и привязки: [АРМ](comps/arm_id.md) и
[Пользователь](comps/user_id.md).

Поля под кнопкой **"Расширенные настройки"** (версия ОС,
[отпечаток железа](comps/raw_hw.md), отпечаток софта) заполняются скриптами
инвентаризации — внесённое руками скрипт, скорее всего, затрёт при следующем
срабатывании.

Выведенную из работы ОС не удаляем, а [архивируем](comps/archived.md).

## Удаление

Удалить (из окна [просмотра](#просмотр) жмем ведро) можно, если

- история об ОС абсолютно точно не нужна, например временно развернутая виртуалка на пару дней
- просто дубликат какой-то ошибочный

В остальных случаях правильнее [архивация](comps/archived.md).
