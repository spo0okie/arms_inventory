# Скрытие технических IP-адресов ОС

Скрипт инвентаризации присылает все адреса сетевых интерфейсов, и среди них
попадаются технические: адреса виртуальных интерфейсов, туннелей (VPN),
кластерные и прочие служебные. В карте адресов ОС их видно, но в учёте они
только мешают.

Такой адрес можно **скрыть значком «глаз»** (перечёркнутый глаз) рядом с
адресом на карточке ОС. Скрытый адрес:

- переносится в список игнорируемых (поле `ip_ignore`);
- **не попадает в реестр IP-адресов**, а значит не участвует в определении
  сетей и [сегментов](../segments.md) ОС;
- **не учитывается в поиске и списках [оборудования](../techs.md)** — раньше
  адреса ОС «подмешивались» к оборудованию, теперь скрытые исключены.

Скрытые адреса по умолчанию свёрнуты. Раскрыть их (вместе с остальными
скрытыми элементами карточки) можно переключателем **«Архивные»**; там же у
каждого скрытого адреса есть значок **«глаз»** для возврата — адрес снова
начнёт учитываться.

Скрытие переживает переинвентаризацию: даже если скрипт снова пришлёт этот
адрес, он останется скрытым, пока его не вернут вручную.
