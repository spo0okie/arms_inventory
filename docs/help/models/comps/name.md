# Операционные системы → Имя компьютера

Имя хоста (без домена — домен указывается отдельным полем). FQDN
(домен+имя) должен быть уникальным; если есть несколько машин
с одинаковым FQDN (клоны, копии из архива), их изолируют
в [песочницах](#doc-anchor:sandbox) — уникальность проверяется в пределах
песочницы.

## Переименование

Если у ОС изменилось имя или домен:

- находим её по старому имени;
- переименовываем и сохраняем;
- если сохранилось без ошибок — готово, дальше можно не читать.

Если при сохранении система сообщает, что такая комбинация домена и имени уже есть, значит в базе есть запись-двойник. Возможны два случая:

- **Старая «мёртвая душа»** — запись ОС, которую когда-то переименовали, а старая осталась. Найдите ОС с новым именем; если она давно не обновлялась — запомните, к какому АРМ она привязана, удалите её, разберитесь, какая ОС на том АРМ на самом деле стоит, и привяжите её.
- **Свежий дубль от скрипта** — если ОС обновлялась совсем недавно, скрипт уже создал запись под новым именем, не привязанную ни к какому АРМ. Просто удалите её — после этого переименование пройдёт без ошибок.
