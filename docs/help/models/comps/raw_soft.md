# Операционные системы → Отпечаток софта

Сырой список ПО, обнаруженного скриптом инвентаризации внутри ОС
(собирается только скриптами Windows-инвентаризации). Руками его не
редактируют — интересна не сама строка, а то, что система из неё
делает: блок **«Софт»** на карточке ОС.

## Категории согласования

Каждая строка отпечатка сопоставляется с продуктом (ПО) и раскладывается
по категориям — сверху вниз от «неинтересных» к требующим внимания:

- **Игнорируемый** — продукт входит в служебный список ПО `soft_ignore`
  (справочник «Списки ПО»: системные компоненты, драйверы и прочий шум).
  Свёрнут; заголовок-тогглер раскрывает список.
- **Бесплатный** — продукт из списка `soft_free`: лицензионный учёт не
  нужен. Тоже свёрнут.
- **Согласованный** — продукт из списка `soft_agreed`: разрешён
  к использованию.
- **Требующий согласования** — распознанный продукт, не входящий ни
  в один из служебных списков. Это рабочая очередь: софт нужно либо
  согласовать, либо запретить/удалить.
- **Не распознанный** — строки отпечатка, которые не удалось сопоставить
  ни с одним продуктом. Кнопка «Создать продукт из этого элемента»
  заводит новый продукт прямо из строки; после появления продукта строка
  при следующем рескане попадёт в одну из категорий выше.

Кнопка «Вписать ПО в паспорт» у распознанного элемента закрепляет продукт
за этой ОС — он попадает в паспорт АРМ.

Если правила распознавания изменились (заведён новый продукт, продукт
добавлен в список), отпечаток пересчитывается не мгновенно — ОС попадает
в очередь рескана (плашка над блоком «Софт» показывает состояние).
