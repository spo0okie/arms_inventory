# Схемы лицензирования

Схемы лицензирования могут очень сильно отличаться даже для одного и того же продукта.
Например:

- Лицензия **Windows Server 2008R2 Standart** допускает запуск 2 виртуальных машин или одной установки на железо. Вроде всё просто, но уже есть нюансы.
- Лицензия **Windows Server 2008R2 Datacenter** использует схему «один экземпляр ОС, запущенный в физ. среде, или неограниченное количество в виртуальной в рамках одной физической машины» — короткого общепринятого названия у этой схемы нет (она тут больше для примера), условно назовём её **«MS Datacenter на хост»**.
- Лицензия **Windows Server 2012R2 Datacenter** похожа на «MS Datacenter на хост», только закупается на количество процессоров (сокетов) физ. сервера, покрывая до 2 сокетов одного сервера на одну лицензию (т.е. нельзя лицензировать 2 односокетных сервера одной лицензией). Допустим, это **«MS Datacenter на сокет»**.
- Лицензия Windows Server 2019 уже лицензируется по ядрам, и т.д.

т.е. есть куча разных нетривиальных схем лицензирования.
А есть куча простых:

- **лицензирование на один ПК** — одна лицензия закрепляется за одним компьютером
- **плавающая лицензия** — несколько лицензий распределяются между бОльшей, чем количество лицензий, группой ПК/пользователей/установок ПО.
- и т.д. и т.п.

Поэтому в зависимости от типа лицензии у неё может использоваться разная схема лицензирования.
Что желательно отмечать в описании схемы лицензирования:

- Какие версии софта она покрывает
  - Возможен ли даунгрейд (до какой версии)
  - Возможно ли обновление ПО (до какой версии)
- Что именно лицензируется? Это могут быть:
  - Пользователи
  - Компьютеры
  - Процессоры (сокеты) или ядра
  - Одновременно запущенные экземпляры

## Список схем

Список доступен через меню **Лицензии => Схемы лицензирования**.
Собственно, кроме названия и описания схемы, в нём ничего особенного и нет, да и быть не должно.

## Редактирование

Тут нужно заполнить:

- **Название** — ну название и название, как идентифицировать такую схему лицензирования
- **Служебное имя** — машинное название схемы латиницей, в нижнем регистре и без пробелов. Пока ни для чего не используется (зарезервировано на будущее)
- **Описание** — пояснение, как работает эта схема лицензирования со всеми важными нюансами, про которые нельзя забывать
- **Ссылки** — ссылки на официальные документы или статьи-пояснения по лицензированию этой схемы
