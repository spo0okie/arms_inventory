# Резервное копирование

Для резервного копирования системы достаточно сохранить файловую структуру и базу данных; специального инструмента не предусмотрено.

Нет никакого специального инструмента, но есть точная информация, что для резервного копирования достаточно **файловой структуры + БД**.
БД можно выгрузить при помощи **mysqldump**.
При резервировании файловой структуры можно игнорировать достаточно тяжелую папку **vendor**, она наполнится автоматически командой

```bash
composer update
```

Главное не потерять:

- локальные конфиги инсталляции: **config/params-local.php**, **config/db-local.php**, **config/web-local.php**, **config/ldap.php**
- загруженные пользователями файлы (сканы документов): **web/scans**

Чтобы потом восстановиться - просто переписываем файловую структуру на место и восстанавливаем БД.

### Docker

При [docker-инсталляции](install-docker.md) все данные и так вынесены из контейнеров наружу: достаточно сохранить папку инсталляции (**db** - файлы БД, **files** - сканы документов, **config** - конфиги). БД надежнее выгружать дампом:

```bash
docker exec -i "arms-arms-db-1" mysqldump -u arms-user -p arms > arms-backup.sql
```
