# Удалённое управление (протокол remotecontrol://)

Значок удалённого подключения (стрелка входа) на карточке ОС — у имени
хоста и у каждого активного IP-адреса — это обычная ссылка вида
`remotecontrol://<fqdn>` или `remotecontrol://<ip>`. Инвентаризация
только формирует ссылку; **что произойдёт по клику — решает обработчик
протокола на компьютере пользователя**. Пока обработчик не установлен,
браузер по клику ничего не сделает (или спросит, чем открыть).

## Готовые обработчики

- [arms-rc-simple](https://github.com/spo0okie/arms-rc-simple) —
  простая схема: reg-файл регистрирует протокол, cmd-скрипт запускает
  ПО удалённого управления (из коробки — Lite Manager, легко заменить
  на своё). Установка: импортировать `register.reg`, положить
  `remotecontrol.cmd` в `C:\Program Files\remotecontrol\`.
- [arms-rc-advanced](https://github.com/spo0okie/arms-rc-advanced) —
  продвинутая схема: управляющее ПО запускается **от другого
  пользователя**, чем браузер, — самому браузеру права удалённого
  управления не нужны. Установка и параметры (`params.vbs`, рабочая
  директория, таймауты) описаны в README репозитория.

## Как написать обработчик самому

Протокол — обычная пользовательская URL-схема Windows:

1. Зарегистрировать схему в реестре (пример — `register.reg`
   из arms-rc-simple): ключ `HKEY_CLASSES_ROOT\remotecontrol` с
   `URL Protocol` и командой запуска в `shell\open\command`.
2. Команда получает первым аргументом весь URL
   (`remotecontrol://host.domain.ru/` — обратите внимание, браузер может
   добавить завершающий слэш). Скрипту остаётся вырезать адрес узла
   и подставить его в командную строку своего ПО удалённого управления
   (Lite Manager, VNC, RDP, `msra` и т.п.).
3. Раздать обработчик на рабочие места администраторов (вручную или
   через GPO).

Схему можно адаптировать под macOS/Linux — там пользовательские
URL-схемы регистрируются средствами desktop-окружения
(`.desktop`-файл с `x-scheme-handler/remotecontrol`).
