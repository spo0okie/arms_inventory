# Generators

## Назначение

Генераторы отвечают за создание значений атрибутов на основе их типа.

```text
attributeData → Generator → value
````

Это низкоуровневый слой генерации данных.

---

## Принципы

### 1. Генераторы должны быть простыми

```text
Generator = type-based value
```

Они:

* не знают бизнес-логики
* не учитывают сложные правила валидации
* не работают с БД
* не создают зависимости между моделями

---

### 2. Генераторы НЕ гарантируют валидность модели

```text
generator → значение
resolver → корректность
factory → итоговая валидность
```

Если модель не проходит `validate()` — это не ошибка генератора.

---

### 3. Минимальная ответственность

Генератор:

* использует `attributeData()['type']`
* может использовать простые параметры (`min`, `max`, `empty`,`nullable`)
* возвращает значение без исключений

---

## Интерфейс

см. `GeneratorInterface`

---

## Ограничения

Запрещено:

* реализовывать `unique`, `exists`
* учитывать связи
* реализовывать бизнес-сценарии
* дублировать логику `ModelFactory` или `Resolver`

---

## Расширение

Кастомные генераторы подключаются в атрибуты через:

```php
'generator' => [
    'class' => CustomGenerator::class,
]
```

---

## Связанные компоненты

```text
params  → входные данные
GeneratorResolver → выбор генератора
ModelFactory      → orchestration
ModelResolver     → исправление модели
```
